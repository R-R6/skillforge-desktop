# SkillForge Desktop v0.1.0 发布说明

> 状态：文档与 CI 配置已就绪；**首次 `v0.1.0` GitHub Release 需在推送 tag 后由 Actions 实跑确认**。  
> 本机已验证：Apple Silicon ARM64 DMG。Windows EXE 与 macOS Intel x64 DMG 仍待 GitHub Actions 首次在线验证。

## 一句话摘要

SkillForge Desktop 首次同时提供 **Windows x64 NSIS**、**macOS ARM64 DMG**、**macOS Intel x64 DMG**。macOS 包为 ad-hoc 临时签名，**未**使用 Apple 开发者证书，也**未**公证。

## 下载选择

| 平台 | 文件名示例 | 适用设备 |
|------|------------|----------|
| Windows 10/11 | `SkillForge Desktop Setup 0.1.0.exe` | x64 |
| macOS Apple Silicon | `SkillForge-0.1.0-macos-arm64.dmg` | M1 / M2 / M3 / M4 等 |
| macOS Intel | `SkillForge-0.1.0-macos-x64.dmg` | Intel Mac |

请勿在 Apple Silicon 上优先选择 x64 包（除非刻意通过 Rosetta 测试）；请勿在 Intel Mac 上使用 arm64 包。

## 本版本变更（相对此前仅 Windows 的桌面版）

### 平台与分发

- 新增 macOS 独立架构 DMG（arm64 / x64），不提供 Universal Binary。
- `npm run package` 继续表示 Windows NSIS；另提供 `package:win`、`package:mac:arm64`、`package:mac:x64`。
- 新增 GitHub Actions：`v*` tag 触发后，三个隔离 job 分别构建，再汇总到同一个 Release。
- macOS 使用 `codesign --sign -` 的 ad-hoc 签名；`mac.identity = null`，禁用证书自动发现与 notarization。

### macOS 运行时

- 补齐标准应用菜单（About / Services / Hide / 编辑 / 窗口等）。
- Dock 点击可恢复或重建窗口；`Cmd+W` 关窗不退出，`Cmd+Q` 退出。
- 托盘使用 Template Image，macOS 单击恢复窗口；Windows 保持双击行为。
- Hermes 在 macOS 优先读 `HERMES_HOME`，否则使用 `~/.hermes`；Windows PowerShell / 进程扫描逻辑仅在 `win32` 执行。

### 构建与校验

- 新增 `scripts/generate-mac-icons.sh`、`scripts/adhoc-sign-mac.sh`、`scripts/after-pack-mac.cjs`、`scripts/verify-mac-dmg.sh`。
- CI / 本地可对 DMG 做签名校验、挂载结构检查、主程序与 `better_sqlite3.node` 架构断言。

## macOS 安装（未公证）

1. 打开 DMG，将 `SkillForge Desktop.app` 拖到“应用程序”。
2. 优先在 Finder 中右键（或 Control 单击）应用，选择“打开”。
3. 若仍被 Gatekeeper 拦截，再执行（通常不需要 `sudo`）：

```bash
xattr -dr com.apple.quarantine "/Applications/SkillForge Desktop.app"
```

这**不代表**应用已通过 Apple 认证。能启动只说明本机允许运行该 ad-hoc 签名包。

## 发版操作清单

1. 确认 `package.json` 的 `version` 为 `0.1.0`，与 tag `v0.1.0` 一致。
2. 提交全部相关改动（含 `resources/icon.icns`、tray Template、scripts、`.github/workflows/release.yml`、文档）。
3. 推送代码后创建并推送 tag：`git tag v0.1.0 && git push origin v0.1.0`。
4. 在 Actions 确认四个 job 成功：`build-windows`、`build-macos-arm64`、`build-macos-x64`、`publish-release`。
5. 在 Releases 页面确认三个安装包均已上传，并检查 Release 正文中的架构与未公证说明。

更细的构建命令与 Gatekeeper 说明见 [build-and-release.md](../build-and-release.md)。
